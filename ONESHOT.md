|------|-----|

| 카카오 채널 | `@inpsytorder` |
| SENDER_KEY (발신프로필 키) | `799de9af7fd86b7301222f39715f012c33d8ed85` |
| 등록 발신번호 | `023305121`, `023367133` (둘 다 사용가능) |

## API 엔드포인트

```
POST https://api2.msgagent.com/api/webshot/send/kakao/AT/inpsyt2
Content-Type: multipart/form-data
```

## 필수 파라미터

| 파라미터 | 값 | 설명 |
|----------|----|------|
| `id` | `inpsyt2` | 원샷 계정 ID |
| `PHONE` | 수신자 번호 (하이픈 없이) | 예: `01012345678` |
| `CALLBACK` | `023305121` | 발신번호 (원샷에 등록된 번호) |
| `MSG` | 템플릿 본문 | 카카오 승인 템플릿과 **글자 하나까지 정확히** 일치해야 함 |
| `SENDER_KEY` | `799de9af7fd86b7301222f39715f012c33d8ed85` | 카카오 발신프로필 키 |
| `TEMPLATE_CODE` | `inpsytorder_paid1` | 원샷 등록 템플릿 코드 |
| `BTN_TYPES` | `웹링크` | **한글**로 입력해야 함. `WL` 쓰면 카카오 에러 21 반환 |
| `BTN_TXTS` | 버튼 텍스트 | 예: `주문내역 확인하기` |
| `BTN_URLS1` | URL | 모바일용 |
| `BTN_URLS2` | URL | PC용 |
| `FAILED_TYPE` | `LMS` | 카카오 미사용자 대체 발송 타입 (**SMS 쓰면 403** — 대체 메시지 길이 초과) |
| `FAILED_MSG` | 대체 문자 본문 | LMS 기준 2000바이트까지 허용 |

## 중요 주의사항

### BTN_TYPES는 반드시 한글
- `WL` 이 아니라 `웹링크` (한글)로 전달해야 함
- `WL`로 보내면 원샷은 result_code 0으로 수락하지만 Kakao가 **에러 21 (잘못된 파라메터 요청21)** 반환
- 버튼 타입 목록: `웹링크`, `앱링크`, `봇키워드`, `메시지전달`, `배송조회`

### FAILED_TYPE은 반드시 LMS
- `SMS`로 설정하면 result_code **403** 반환 (대체메시지 길이 제한 초과)
- 이유: 이벤트명 + URL이 포함된 메시지가 SMS 90바이트 제한을 넘음
- LMS는 2000바이트까지 허용 → 항상 `LMS` 사용

### MSG는 승인된 템플릿과 정확히 일치
- 카카오 비즈센터에서 승인받은 템플릿 본문과 변수 치환 결과가 완전히 동일해야 함
- 공백, 줄바꿈 하나라도 다르면 발송 실패

### HTTP 200이어도 실패일 수 있음
- API는 항상 HTTP 200 반환
- 응답 body의 `result_code`를 반드시 확인해야 함
- **성공: `result_code === 0`**
- 그 외 모든 값은 실패

### Supabase Edge Function(Deno)에서는 직접 호출 불가
- msgagent.com 서버가 구형 TLS를 사용 → Deno 런타임에서 HandshakeFailure 발생
- 반드시 **프론트엔드(브라우저)에서 직접 fetch** 해야 함

## HTML 파일로 직접 테스트하는 법

원샷이 제공한 `카카오연동테스트/api연동테스트.html` 파일을 사용합니다.

1. 파일을 브라우저로 열기 (Chrome 권장)
2. 각 필드를 아래 값으로 채우기:

| 필드 | 값 |
|------|----|
| 사용자 아이디 | `inpsyt2` (이미 입력됨) |
| 전송 아이디 | 비워도 됨 |
| 수신 번호 | 테스트할 휴대폰 번호 (하이픈 없이) |
| 발신 번호 | `023305121` |
| 전송 시간 | 비워도 됨 (즉시발송) |
| 발송 메시지 | `홍길동님, 안녕하세요.\n2026 테스트에서 결제가 완료되었습니다.\n\n주문 내역은 아래에서 확인하실 수 있습니다.` |
| 카카오 발신 프로필 키 | `799de9af7fd86b7301222f39715f012c33d8ed85` (이미 입력됨) |
| 카카오 알림톡 템플릿 코드 | `inpsytorder_paid1` (이미 입력됨) |
| 카카오 대체 메시지 타입 | `LMS` |
| 카카오 대체 메시지 제목 | `인싸이트 주문 알림` |
| 카카오 대체 메시지 | `[인싸이트] 홍길동님, 결제가 완료되었습니다.` |
| 카카오 버튼 타입 리스트 | `웹링크` |
| 카카오 버튼 이름 리스트 | `주문내역 확인하기` |
| 카카오 버튼링크주소 URL1 | `https://inpsytorder.vercel.app/order/status/테스트토큰` |
| 카카오 버튼링크주소 URL2 | `https://inpsytorder.vercel.app/order/status/테스트토큰` |

3. **전송** 버튼 클릭
4. 브라우저 화면에 응답 JSON이 표시됨 — `result_code: 0`이면 성공

> **주의:** HTML 파일의 `SEDNER_KEY` 필드명은 개발자 오타. 실제 API 파라미터는 `SENDER_KEY`가 맞음. HTML 폼은 어차피 `name` 속성 그대로 전송되므로 테스트할 때 결과가 다를 수 있음. 테스트 결과가 이상하면 `name` 속성을 `SENDER_KEY`로 수정 후 사용.

## 현재 사용 중인 템플릿

**템플릿 코드:** `inpsytorder_paid1`

**본문:**
```
{고객명}님, 안녕하세요.
{이벤트명}에서 결제가 완료되었습니다.

주문 내역은 아래에서 확인하실 수 있습니다.
```

**버튼:** 주문내역 확인하기 (웹링크) → `https://inpsytorder.vercel.app/order/status/{access_token}`

**발송 조건:**
- `status` → `paid` 변경 시
- `is_on_site_sale = true`이면 발송 안 함 (현장수령 주문)
- `phone_number`가 없으면 발송 안 함
