# 09. 가시성 세이프가드

> 우리는 별도의 다크 모드를 만들지 않는다. 고대비 토글도 큰글자 토글도 만들지 않는다(D10). 그렇다고 우리 화면이 "라이트 모드 한 종류"만 산다고 말하기엔, 사용자 쪽 OS가 가만히 있어주지 않는다. 누군가는 iOS Smart Invert를 켜고 들어오고, 누군가는 Windows 고대비 모드를 켠 채로 어드민을 열고, 누군가는 Android Force Dark가 항상 켜져 있다. 이 문서는 **우리가 다크 테마를 만들지 않는다는 결정을 유지하면서도, OS가 강제로 색을 바꿔 끼우는 상황에서 우리 화면이 안 깨지게** 만드는 방법에 대한 글이다.

## 어느 의사의 검색 박스 이야기

먼저 실제로 겪은 일부터. 어느 학회에서 50대 의사 한 분이 본인 폰의 OS 다크 모드를 항상 켜놓고 다닌다. 그분이 우리 주문서를 열었다. 상품 리스트 위쪽에 검색 박스가 하나 있는데, 그분 화면에서는 **흰 글씨가 흰 배경에 떠 있어서 본인이 뭘 입력하는지 보이지 않았다.**

우리는 다크 모드를 만들지 않았다. 그러면 왜 흰 배경에 흰 글씨가 됐을까. 답은 우리 쪽 색이 아니라 **OS와 브라우저가 폼 요소에 자기 색을 끼워넣었기 때문**이다.

`input`, `select`, `textarea` 같은 폼 요소는 페이지 어디서든 OS의 영향을 강하게 받는다. iOS에서 Smart Invert가 켜져 있으면 흰 배경은 검정으로 뒤집히고, 글씨 색은 그대로 검정으로 두기 때문에 결과적으로 검정 배경에 검정 글씨가 된다. Android에선 Force Dark가 비슷한 일을 한다. 우리가 CSS로 `background: white; color: black`이라고 명시해도, **OS는 그 위에 한 번 더 자기 규칙을 덧씌운다.**

검색 박스 문제의 메커니즘은 그것이었다. 우리 잘못이 아니라고 말할 수도 있지만, 그 의사 입장에서는 "이 사이트가 안 보인다"이다. 그래서 우리는 책임지기로 했다.

## 처리 정책 세 가지

### 1. color-scheme — OS 강제 다크의 99%는 여기서 막힌다

우리 페이지의 폼 요소들에 한 줄을 박는다 — `color-scheme: light`. 이게 무슨 말이냐면 "이 요소는 라이트 모드 기준으로만 색을 받아라, OS가 다크 모드여도 끼어들지 마라"는 선언이다.

이 한 줄로 iOS Smart Invert, Android Force Dark, macOS·Windows 다크 모드 강제 상황에서 **검색 박스·텍스트 입력·드롭다운이 자기 색을 유지**한다. 검색 박스에 흰 글씨가 떠서 안 보이는 사고는 여기서 끝난다.

페이지 전체에 걸 수도 있고, 위험한 컴포넌트(`input`·`select`·`textarea`·검색 박스)만 골라서 걸 수도 있다. 우리는 **페이지 전체(`html`·`body`)에 라이트 선언**, **그리고 폼 요소에 한 번 더 선언**한다. 이중으로 거는 이유는 일부 OS·브라우저 조합에서 페이지 단위 선언이 무시되는 케이스가 있어서.

### 2. forced-colors — Windows 고대비 모드

Windows에는 "고대비 모드"라는 OS 옵션이 있다. 시각 보조가 필요한 사용자가 켠다. 켜진 순간 **우리 색은 거의 다 의미가 없어진다.** OS가 텍스트는 한 가지 색, 배경은 한 가지 색, 버튼은 한 가지 색, 링크는 한 가지 색으로 강제로 바꿔치기한다.

이걸 감지하는 미디어 쿼리가 `@media (forced-colors: active)`다. 이 안에서는 우리 색을 고집하지 않는다. 대신 **보더와 아이콘**을 OS의 시스템 키워드(`Canvas`, `CanvasText`, `ButtonText`, `Highlight` 같은 것)로 fallback 시켜둔다. 그러면 OS가 자기 색을 끼워넣을 때 보더가 사라지지 않고, 아이콘이 배경에 묻히지 않는다.

여기서 우리가 보장하는 건 **"요소가 어디에 있는지 알아볼 수 있다"**까지다. 우리 브랜드 청록색이 그대로 보일 거라고 약속하지 않는다. 그건 OS의 영역이고, 우리가 깨려는 것도 아니다.

### 3. prefers-contrast — 일반 고대비 선호자

OS 고대비 모드를 켜는 사람과는 다르게, **"일반 색 시스템 안에서 좀 더 또렷한 대비를 선호한다"**고 OS에 알린 사용자가 있다. iOS의 "대비 증가", macOS의 "대비 증가", Windows의 일부 설정이 여기 해당된다.

이 사람들에게 우리는 **회색 톤을 한 단계 진하게** 보여준다. `surface-3`은 `surface-4` 쪽으로, 보조 텍스트는 본문 톤 쪽으로. 보더 굵기도 한 단계 굵게. 본문 대비를 7:1 위로 끌어올린다. 이게 `@media (prefers-contrast: more)` 블록에서 일어나는 일이다.

이건 OS 강제가 아니다. 우리가 우리 시스템 안에서 한 단계 더 친절해지는 선택이다. 그래서 다크 모드와 다르게 **별도 테마가 아니라 같은 토큰의 값 미세 조정**이다.

## 그래서 우리는 어디까지 보장하는가

이 세 가지를 깔아두고 나서, 우리는 다음과 같이 말할 수 있다.

- OS가 다크 모드라도, 우리 화면은 라이트 색을 유지한다. 폼 요소도 마찬가지.
- iOS Smart Invert·Android Force Dark가 켜져 있어도, 검색 박스가 안 보이지 않는다.
- Windows 고대비 모드 사용자에게도, 요소의 위치와 경계가 흐려지지 않는다.
- "대비 증가"를 켠 사용자에게는 회색이 한 단계 진해진다.

반대로 보장하지 **않는** 것도 분명히 적는다.

- 다크 모드 사용자에게 다크 테마 디자인을 제공하지 않는다. 라이트 색을 그대로 유지한다.
- Windows 고대비 모드에서 우리 브랜드 색이 그대로 보일 거라고 약속하지 않는다. OS가 색을 바꾸면 그건 OS가 책임진다.
- 시스템 줌 200% 이상에서 레이아웃이 그대로 유지될 거라 약속하지 않는다. 150%까지가 우리 책임선이다.
- 음성 스크린리더 친화 라벨링은 베타 단계에서 보장하지 않는다. M4 이후 다음 주기 작업이다.

핵심은 이거다 — **우리는 강제 모드 사용자에게도 우리 디자인을 100% 그대로 보여줄 의무가 없다. 우리가 막는 건 "요소가 안 보이는 사태" 그 하나다.**

## 어디에 코드가 들어가는가

세 가지 처리는 모두 글로벌 CSS 한 곳에서 다룬다. 페이지마다 개별 입력 컴포넌트에서 따로 처리하지 않는다. 이렇게 하지 않으면 새 페이지를 만들 때마다 누군가 빼먹는다.

구체적으로는 `src/styles/`(또는 `src/index.css`) 같은 글로벌 CSS 진입점에 세 블록이 들어간다. CSS 변수로 정의된 토큰 위에 `@media (prefers-contrast: more)` / `@media (forced-colors: active)`가 같은 변수에 다른 값을 할당하고, 폼 요소 셀렉터에 `color-scheme: light`가 박힌다. 토큰 값의 정확한 매핑은 `Appendix_Spec.md` §11에 정리.

## 게이트 3 검수에서 확인할 것

실기기 QA(게이트 3) 시점에 아래를 본다.

- iOS Safari 다크 모드 켠 상태로 모든 페이지가 정상 표시되는가
- iOS Smart Invert 켠 상태로 주문서 검색 박스·입력 필드가 안 보이지 않는가
- Android Chrome 다크 모드·Force Dark 켠 상태로 동일
- Windows 고대비 모드 켠 상태로 어드민 주문 관리에서 보더·아이콘이 사라지지 않는가
- iPad 글자 크기 큼/매우큼 설정으로 주문서가 끊기지 않는가
- 데스크탑 크롬 줌 150%에서 어드민 한 화면 안에 잘리지 않는가
- `prefers-contrast: more` 켜고 텍스트 대비 7:1 통과

## 무엇을 하지 않는가 (재확인)

- 다크 모드 전용 디자인을 따로 만들지 않는다.
- 다크 모드 토글·고대비 토글·큰글자 토글을 어디에도 두지 않는다.
- OS 강제 모드 사용자에게 우리 색 그대로를 약속하지 않는다.

이 결정은 "유저가 원할 수도 있지 않을까" 하는 의심이 들어도 유지한다. 그 의심은 1만 명이 매일 쓰는 서비스에서 의미가 있고, 우리에겐 의미가 없다. 우리는 **연 800건, 1년에 8일** 쓰이는 서비스다. 그 규모에서 우리가 해야 할 일은 **OS 강제로 깨지는 일을 막는 것**까지다.
