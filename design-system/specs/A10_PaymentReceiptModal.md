# A10b — 지불증(수당 영수증) 생성 모달 · 기획 확정

> 학회 종료 후 참석 직원 수당(주말근무·출장비) 영수증을 ExcelJS 양식으로 다운로드.
> 입금결의서(매출 정산) 패턴 확장 — 인건비 정산용 두 번째 산출물.
> 상태: **기획 확정.** 착수 전제 = **A10 P0(`attendee_ids` 포함) 머지 후.**

## 1. 진입·위치
- **A10 목록 행 `⋯` 메뉴 → "지불증 내보내기"** (L2 미구현이라 L1 ⋯메뉴 진입. 입금결의서 "행사 선택→버튼" 선례).
- 클릭 → 모달 1개로 완결(위저드 금지).

## 2. 수당 규정 (고정 단가)
- **반일 근무 = 40,000원** (오전 09~13/09~14/10~14/10~15, 오후 13~17 — 시간대·식사 무관 40,000 동일)
- **종일 근무 = 70,000원** (09~17 또는 10~18, 식사 자유)
- **출장비 = 20,000원 / 평일 1박당** (지방 학회 시)
- **시간대 규정 = `WEEKEND_SLOTS`(allowanceRules.js) 단일 소스.** 주말 블록은 `slotId`로 시간대를 보유하고, 단가는 선택 slot의 `rate`로 산출(반일 slot=40,000 / 종일 slot=70,000). `timeMode` 토글 폐기(2026-06-08 정정).

  | id | 시간 | 식사 | 수당 | 구분 |
  |---|---|---|---|---|
  | am0913 | 09:00~13:00 | X | 40,000 | 반일 |
  | am0914 | 09:00~14:00 | O | 40,000 | 반일 |
  | am1014 | 10:00~14:00 | X | 40,000 | 반일 |
  | am1015 | 10:00~15:00 | O | 40,000 | 반일 |
  | pm1317 | 13:00~17:00 | X | 40,000 | 반일 |
  | full0917 | 09:00~17:00 | (자유) | 70,000 | 종일 (기본) |
  | full1018 | 10:00~18:00 | (자유) | 70,000 | 종일 |
- **중복방지(정정): 같은 날짜(member,date)에 주말근무 + 출장 동시 지급 불가.**
  - ✅ 한 멤버가 **금(출장 1박) + 토·일(주말근무)** 를 한 지불증에 함께 받는 것은 **정상**(날짜가 다름).
  - ✅ 하나의 지불증에 **주말근무 블록 + 출장 블록 공존 가능**, 멤버가 양쪽에 등장 가능.
  - ❌ 같은 날짜에 둘 다 선택 = 차단(하나만).

## 3. 입력 단위 = 「날짜/항목 블록」 (확정 — 축 전환 2026-06-08)
> 기존 "멤버 × 날짜 셀" 모델 폐기. **엑셀 양식 블록과 1:1 매핑되는 「날짜/항목 블록」** 모델로 전환.
> 근거: 양식이 주말근무 블록(A8 날짜·B8 시간라벨·B9~11 인원·C9~11 수당) + 출장비 블록(A14·B14·B15~17·C15~17) 구조라, 입력 축을 양식 블록에 맞추면 export 동적행 생성과 직결되고 사용자 멘탈모델도 자연스럽다.

- **블록 = 양식 블록과 1:1.** 모달 본문 = 「항목 블록」 리스트 + 하단 `+ 항목 추가`(주말출근비 / 출장비 선택).
- **주말출근비 블록:**
  - 날짜 = **여러 날 다중선택**(예 10/18·10/19). 시간대 = `반일 40,000 / 종일 70,000` (블록 전체 공통, 토글).
  - 인원 리스트 + `+ 인원 추가`. **1인 수당 = 단가 × 선택 날짜 수.** 블록 총합 = 1인 수당 × 인원 수.
- **출장비 블록:**
  - 날짜 = **범위 선택(시작~종료)**, 라벨 "10/16(목)~10/17(금)". **박수 = 범위에서 자동(박수 = 종료−시작 일수)**. 박수 직접입력 없음.
  - 인원 리스트 + `+ 인원 추가`. **1인 수당 = 20,000 × 박수.** 블록 총합 = 1인 수당 × 인원 수.
- **한 인원이 여러 블록에 등장 가능**(예: 금 출장 + 토·일 주말 = 출장 블록 + 주말 블록 양쪽).
- **중복 검증(§2):** 같은 인원이 **같은 날짜**에 주말 블록 + 출장 블록 양쪽에 있으면 경고(같은날짜 중복지급 금지). 다른 날짜는 정상.
  - 단일 셀 선택이 아니므로 구조적 원천차단 불가 → **명시적 검증 + 경고 UI 필요**(인원 행 빨간 표시 + 상단 배너 + 다운로드 차단).

## 4. 모달 구조 (날짜/항목 블록)
- **상단:** 학회명/행사명·기간·장소(읽기전용 카드).
- **중복 경고 배너:** 중복 인원 존재 시 상단에 적색 배너(N명 + 해결 전 다운로드 불가 안내).
- **블록 리스트:** 각 블록 카드 = [헤더: 아이콘·종류·단가요약·블록합산·삭제] + [날짜 컨트롤(다중/범위) + 주말은 시간대 토글] + [인원 리스트 + 인원 추가].
  - **인원 추가:** 참석자 후보(`attendee_ids` join, 직급 자동, 정렬 직급순>이름순) **멀티선택** + **참관 직원**(이름 입력 + 직급 드롭다운 차장/과장/대리/사원). 참관은 **저장 안 함(1회성)**, 주황 "참관" 배지.
  - 인원 각자 옆에 1인 수당·삭제. 중복 인원은 행 빨간 보더 + "날짜 중복" 경고.
- **하단:** **총합(숫자)+한글금액**(`금 {numberToKoreanCurrency}원정`) 실시간 + 지급 인원 수(유니크). 영수인=로그인 관리자(읽기). 다운로드 버튼.
- 모달 1개 완결(위저드 X). 모바일(sm↓) fullScreen.

## 6. ExcelJS 출력 (지불증.xlsx)
- 시트: `03.지불증 (템플릿)` 채움 / `03.지불증 (참고)` 제거(입금결의서 동일). 양식은 `public/templates/`에 배치(첨부 파일 확보됨).
- **동적 행:** 멤버 N명 + **날짜별 블록 N개**(주말근무 블록은 근무일마다 1개 + 출장 블록) → `spliceRows`/`insertRow('i+')` 서식 상속 삽입.
  - ⚠️ 난점: 행 삽입 시 총합(B4/D4)·병합셀 위치 밀림 → **아래 블록부터 역순 삽입 + 총합 셀 재탐색**(`getRow`).
- 금액=규정엔진 계산(`allowanceRules.js` 순수함수), 영수인=로그인 관리자(`profile.name`), 작성일=오늘.
- **셀 매핑 (2026-06-08 정정):**
  - **B4 = `numberToKoreanCurrency(total) + '원정'`** (예 "사십육만원정"). 양식 A4의 "금"과 합쳐 "금 사십육만원정".
  - **D4 = `total` (숫자)** (예 460000). (B4·D4 맞바꿈 + B4에 '원정' 추가 — 기존 B4=숫자/D4=한글에서 정정.)
  - **B8 = `"{slot.start} ~ {slot.end} 주말 근무"`** (예 "09:00 ~ 17:00 주말 근무"). 선택 시간대 라벨. 기존 "종일"/"반일" 폐기. 다중 주말 블록 병합 시 시간대가 다르면 `;`로 병기(단일 블록이 일반).

## 7. 데이터·검증
- 인원 = `attendee_ids` join(직급 자동) + 참관 ad-hoc(이름·직급 문자열, 미저장).
- 규정엔진(`allowanceRules.js`): 주말블록 1인=단가(반일40k/종일70k)×날짜수 / 출장블록 1인=20k×박수(범위 종료−시작). 블록합=1인×인원수. 총합=Σ블록합.
- **중복검증:** 인원별 주말날짜집합 ∩ 출장날짜집합 ≠ ∅이면 충돌(같은날짜 중복지급 금지, §2). 충돌 시 다운로드 차단.
- QA: 금액·총합·한글 정확성 / 같은날짜 중복검출·차단 / 인원 양블록 등장 / N블록·N인원 양식 행 무결성 / 참관 추가 / 빈상태·전원0원 엣지 / **다운로드 파일 셀 실측**(jsdom 미검증).

## 9. 모달 시안 (디자인 확정 — 2026-06-08, 축 전환판)
> Preview: `/preview/payment-receipt` · 컴포넌트 `src/components/PaymentReceiptModalPreview.jsx` (mock, 미연결).
> 합성 컴포넌트(SectionCard/EmptyState/PageHeader) 재사용. theme.js 미수정.
> **축 전환:** 기존 "멤버 카드 중심"(멤버×날짜 4세그먼트) → **"날짜/항목 블록 중심"**. 양식 블록과 1:1·export 직결·멘탈모델 자연스러움.

### 9-1. 레이아웃 결정 = **항목 블록 카드 리스트**
- 모달 본문 = 「항목 블록」 카드 세로 스택 + 하단 `+ 항목 추가` 메뉴(주말출근비 / 출장비).
- **블록 카드 1장** = [헤더: 종류 아이콘·라벨·단가요약·블록합산·삭제] + [날짜 컨트롤 + (주말)시간대 토글] + [인원 리스트 + 인원 추가].
- **멤버 중심 모델 폐기 근거:** 멤버×날짜 셀은 "한 인원이 토=출장·일=주말" 같은 혼합을 셀 단일선택으로 표현했으나, 양식이 **블록 단위**(주말근무 블록 / 출장 블록)라 export 시 멤버축→블록축 재구성이 필요했음. 블록 중심이면 양식 동적행 삽입과 1:1.

### 9-2. 주말출근비 블록
- **날짜 = 다중선택**(경량 캘린더 팝오버, 여러 날 토글). 라벨 "10.18(일), 10.19(월)".
- **시간대 = 규정 드롭다운**(`WEEKEND_SLOTS` 7개, §2 표) = 블록 전체 공통. MUI Select, `ListSubheader`로 반일·40,000 / 종일·70,000 그룹. 옵션 = "{시간} · 식사 O/X(반일) / 식사 자유(종일)", 선택 표시 = "09:00~17:00 · 종일 70,000". hit-area 44/48px. 기본 slot = full0917. (반일/종일 토글 폐기 — 2026-06-08 정정.)
- 인원 리스트. **1인 수당 = 선택 slot.rate × 선택 날짜 수** (헤더 "종일 70,000 × 2일", 인원행 옆 1인 금액).

### 9-3. 출장비 블록
- **날짜 = 범위선택**(캘린더 팝오버, 시작→종료 2클릭). 라벨 "10.16(목)~10.17(금)".
- **박수 자동** = 종료−시작 일수(직접입력 없음). 헤더 "20,000 × 1박".
- 인원 리스트. **1인 수당 = 20,000 × 박수.**

### 9-4. 인원 추가 = 참석자 멀티선택 + 참관 직원
- 인원 추가 팝오버: 상단 **참석자 후보 멀티선택**(`attendee_ids` join, 직급 자동, 정렬 직급순>이름순, 체크박스 + N명 추가) / 하단 **참관 직원**(이름 입력 + 직급 드롭다운, "저장 안 됨·1회성" 명시).
- 인원 행: 이름·직급 칩·(참관=주황 배지)·1인 수당·삭제. 같은 인원이 여러 블록 등장 정상.

### 9-5. 중복 검증 (같은 인원·같은 날짜 주말+출장)
- **블록 단위 모델은 단일선택 원천차단이 없으므로 명시적 검증 필요**(§2). `computeConflicts`: 인원별 주말 날짜집합 ∩ 출장 날짜집합 ≠ ∅이면 충돌.
- UI 3단: ① 충돌 인원 행 = error 보더+배경+"날짜 중복" 칩(툴팁에 충돌 날짜) ② 상단 적색 배너(N명, 해결 전 다운로드 불가) ③ 다운로드 disabled + 툴팁.

### 9-6. 하단 = 총합 미리보기 + 다운로드
- **총 지급액(h4 brand) + 한글금액**(`금 {numberToKoreanCurrency}원정`) 실시간. ⚠️ 헬퍼는 순수 한글숫자만(‘원’ 미포함) → 화면도 ‘금…원정’ 래핑(양식 A5/M5 컨벤션 일치).
- **지급 인원 N명 = 유니크**(여러 블록 중복 카운트 방지, 1인 수당>0 블록 소속만).
- 영수인 = 로그인 관리자(읽기, `profile.name`+직급). **다운로드 비활성:** 빈상태 / 전원 0원 / **중복 미해결**.

### 9-7. 캘린더 = 자체 경량 팝오버 (의존성 주의)
- ⚠️ `@mui/x-date-pickers` **미설치**. 시안은 date-fns 미사용 순수 JS 경량 캘린더(다중/범위)로 구현 → **실서비스 라이브러리 도입 여부는 frontend-engineer 판단**(후속 요청).
- 다중 모드: 날짜 토글 + 완료. 범위 모드: 시작→종료 2클릭, 사이 칸 음영. 주말 적색 토큰.

### 9-8. AI 시그니처 차단 자가점검 (룰 E)
- 그라데이션·가짜 통계·의미없는 좌/우 컬러바 **없음**. 색은 모두 의미 토큰(primary=선택/합계, error=주말·중복·주말블록, info=출장블록, attention=참관 배지, gray=중립).
