# A10b — 지불증(수당 영수증) 생성 모달 · 기획 확정

> 학회 종료 후 참석 직원 수당(주말근무·출장비) 영수증을 ExcelJS 양식으로 다운로드.
> 입금결의서(매출 정산) 패턴 확장 — 인건비 정산용 두 번째 산출물.
> 상태: **기획 확정.** 착수 전제 = **A10 P0(`attendee_ids` 포함) 머지 후.**

## 1. 진입·위치
- **A10 목록 행 `⋯` 메뉴 → "지불증 내보내기"** (L2 미구현이라 L1 ⋯메뉴 진입. 입금결의서 "행사 선택→버튼" 선례).
- 클릭 → 모달 1개로 완결(위저드 금지).

## 2. 수당 규정 (고정 단가)
- **반일 근무 = 40,000원** (오전 09~13/09~14/10~14/10~15, 오후 13~17 — 시간대·식사 무관 40,000 동일)
- **종일 근무 = 70,000원** (09~17 또는 10~18, 식사 자유)
- **출장비 = 20,000원 / 평일 1박당** (지방 학회 시)
- **중복방지(정정): 같은 날짜(member,date)에 주말근무 + 출장 동시 지급 불가.**
  - ✅ 한 멤버가 **금(출장 1박) + 토·일(주말근무)** 를 한 지불증에 함께 받는 것은 **정상**(날짜가 다름).
  - ✅ 하나의 지불증에 **주말근무 블록 + 출장 블록 공존 가능**, 멤버가 양쪽에 등장 가능.
  - ❌ 같은 날짜에 둘 다 선택 = 차단(하나만).

## 3. 입력 단위 = 멤버 × 날짜 (확정 b)
- 같은 멤버가 **날짜별로 다른 패턴**(토=출장, 일=종일근무) 가능 → (멤버, 날짜) 셀 단위 입력.
- 각 (멤버, 날짜) 셀 = `미근무 / 반일(40k) / 종일(70k) / 출장(20k·평일1박)` 중 1택(단일 선택이라 같은 날짜 중복 원천 차단).
- 날짜 후보 = 학회 기간(start~end) + **출장 전날(평일)** 포함 가능 → 모달에서 날짜 추가 허용.

## 4. 모달 구조 (멤버×날짜 입력)
- **상단:** 학회명/행사명(읽기전용) + 근무 날짜 목록(학회 기간 자동 + "날짜 추가"로 평일 출장일 보강).
- **멤버 행 리스트:** 정렬 **직급순→이름순**. 각 멤버 × 각 날짜 셀에 위 1택.
  - 참석자 자동 표시(`attendee_ids` → `user_profiles` join, 이름·직급).
  - **"+ 참관 직원 추가"**: 이름·직급(드롭다운) 직접 입력 ad-hoc 행(미등록자). **저장 안 함(1회성)**.
  - 미근무 멤버는 금액 0 → 문서에서 제외.
- **하단:** 멤버별 합산 → **총합(숫자)+한글금액**(`numberToKoreanCurrency`) 실시간 미리보기 → "지불증 다운로드".
- (구현 시 레이아웃: 멤버×날짜 그리드 vs 날짜별 섹션 — 빌드 시점 디자인 확정. 셀 단일선택만 지키면 됨.)

## 5. 직급(`position`) — 확정
- **`user_profiles`에 직급 컬럼 신설**(department 추가 20260602 선례 답습). **Dooray API 연동 기각**(연 8일에 외부 동기화 부채=과설계).
- **고정 드롭다운(자유입력 금지)** — 표기흔들림 방지·정렬 보장.
- **직급 목록·정렬순서: 차장 > 과장 > 대리 > 사원** (4단). 참관직원도 동일 드롭다운.
- 사용자관리 프로필 모달에 입력칸 추가(name/department 패턴 복제). list-users select에 포함.

## 6. ExcelJS 출력 (지불증.xlsx)
- 시트: `03.지불증 (템플릿)` 채움 / `03.지불증 (참고)` 제거(입금결의서 동일). 양식은 `public/templates/`에 배치(첨부 파일 확보됨).
- **동적 행:** 멤버 N명 + **날짜별 블록 N개**(주말근무 블록은 근무일마다 1개 + 출장 블록) → `spliceRows`/`insertRow('i+')` 서식 상속 삽입.
  - ⚠️ 난점: 행 삽입 시 총합(B4/D4)·병합셀 위치 밀림 → **아래 블록부터 역순 삽입 + 총합 셀 재탐색**(`getRow`).
- 금액=규정엔진 계산(`allowanceRules.js` 신규 순수함수), 총합 B4/D4 + 한글금액, 영수인=로그인 관리자(`profile.name`), 작성일=오늘.

## 7. 데이터·검증
- 멤버 = `attendee_ids` join(직급 자동) + 참관 ad-hoc(이름·직급 문자열, 미저장).
- 규정엔진: (멤버,날짜)→금액 매핑, 같은날짜 단일선택 강제, 출장 20k×평일박수.
- QA: 금액·총합·한글 정확성 / 같은날짜 중복차단 / 멤버 양블록 등장 / N명·N날짜 양식 행 무결성 / 참관 추가 / 전원미근무·참석자0 엣지 / **다운로드 파일 셀 실측**(jsdom 미검증).

## 8. 빌드 순서·위임
1. **(선행) A10 P0 머지** — `attendee_ids` 필요.
2. backend: `user_profiles.position` 컬럼 + update-user-profile/list-users 확장.
3. frontend: 사용자관리 직급 입력칸 + `allowanceRules.js`(+테스트) + 지불증 export 유틸(다중 date-block 동적행) + ⋯메뉴 모달.
4. qa-pm: 계산·중복·정합 QA + 실데이터 export 셀 실측.

## 9. 모달 시안 (디자인 확정 — 2026-06-08)

> EventHubListPreview 톤·합성 컴포넌트(SectionCard/EmptyState/PageHeader) 재사용. theme.js 미수정.

### 9-1. 레이아웃 결정 = **멤버별 날짜 서브행** (그리드 기각)
- **채택:** 멤버 카드 1장 = [헤더: 이름·직급·(참관)·멤버합산·삭제] + [본문: 날짜별 1줄 × 4세그먼트 단일선택].
- **그리드(멤버행 × 날짜열, 셀=드롭다운) 기각 근거:**
  1. 3~5명×3~4일 = 셀 12~20개. 셀 폭이 좁아 "출장 20,000" 단가 라벨이 안 들어감 → 의사·연구자 대상 "정확성·신뢰" 원칙 위배(룰 §설계원칙).
  2. 셀이 작은 드롭다운이면 "지금 무엇이 선택됐나"가 한눈에 안 보임. 학회 직후 서두르는 입력 맥락에 부적합.
  3. 멤버별 합산을 행 끝에서 즉시 보여주려면 열이 하나 더 필요 → 가로 폭 압박.
- **서브행 장점:** ① 멤버 단위 합산이 카드 헤더에 항상 노출(검산 쉬움) ② 각 날짜 줄이 가로 full-width라 4세그먼트에 "라벨+단가" 동시 표기 가능 ③ 멤버 추가/제거가 카드 단위 = 직관적 ④ 모바일 fullScreen에서도 세로 스택으로 자연 적응(고객 화면 아니지만 태블릿 세로 대비).

### 9-2. (멤버,날짜) 입력 = 4세그먼트 단일선택
- 한 줄 = 한 날짜. `미근무 / 반일 40,000 / 종일 70,000 / 출장 20,000` 중 **하나만** 선택(segmented control, radiogroup).
- **단일선택 = 같은 날짜 주말근무+출장 동시지급 차단을 구조적으로 보장**(§2 정정 규정). 별도 검증 경고 불필요.
- 세그먼트 hit-area 52px(터치). 선택 시 흰 배경+brand 보더+단가 강조. 미근무는 회색 보더(0원 = 문서 제외 신호).
- 주말 날짜는 요일 라벨 적색 토큰(error.main)으로 시각 구분(주말근무 맥락 암시, 강제 아님).

### 9-3. 상단 = 학회(읽기) + 근무 날짜
- 학회명·기간·장소 읽기전용 카드. 근무 날짜는 기본 = 학회 start~end 자동.
- **"날짜 추가"**: 평일 출장 전날 보강용. 후보를 메뉴로 제시(평일=‘출장 전날 보강용’ 보조라벨). 날짜는 정렬 유지.

### 9-4. 멤버 = 참석자 자동 + 참관 추가
- 참석자 = `attendee_ids` join, 정렬 **직급순(차장>과장>대리>사원)→이름순**. 직급 칩(회색) 표기.
- **"+ 참관 직원 추가"**: 인라인 폼(이름 입력 + 직급 드롭다운). adhoc 행, 주황 "참관" 배지 + 삭제 버튼. **저장 안 됨(1회성)** 명시 문구 노출.
- 멤버 합산 0원이면 카드 회색 음영(미근무 = 문서 제외 신호). 합산은 헤더 우측 number 위계.

### 9-5. 하단 = 총합 미리보기 + 다운로드
- **총 지급액(숫자, h4 brand) + 한글금액**(`금 {numberToKoreanCurrency}원정`, 예 26만원 → "금 이십육만원정") 실시간.
  - ⚠️ 한글 헬퍼는 순수 한글 숫자만 반환(‘원’ 미포함). 양식 셀이 ‘금/원정’을 감싸므로 화면 미리보기도 동일 규칙으로 ‘금…원정’ 래핑.
- 지급 대상 N명(합산>0 멤버 수) 보조 표기. 영수인 = 로그인 관리자(읽기, `profile.name`+직급).
- **다운로드 비활성 조건:** 참석자 0(빈상태) / 전원 미근무(총합 0). 각각 EmptyState·안내문으로 다음 행동 제시(룰 원칙4).

### 9-6. 엣지 처리
- **참석자 0:** EmptyState("참석자가 없습니다") + "참관 직원 추가" CTA → 추가하면 일반 입력 흐름 진입.
- **전원 미근무(0원):** 총합 패널 회색 + "근무 항목을 하나 이상 선택" 안내, 다운로드 disabled.
- 모달 1개 완결(위저드 X). 모바일(sm↓) fullScreen.

### 9-7. AI 시그니처 차단 자가점검 (룰 E)
- 그라데이션·가짜 통계·의미없는 좌/우 컬러바 **없음**. 색은 모두 의미 토큰(primary=선택/합계, error=주말, attention=참관 배지, gray=미근무 음영).

### 9-8. 빠진 관점 / 확인 질문 → §확인 필요
- (Q1) 한글금액 표기 = "금 …원정" 래핑이 양식(A5/M5) 컨벤션과 화면에서 일치하는가? (depositResolution 패턴 답습 전제)
- (Q2) "참관 직원" 명칭이 현장 용어로 맞는가(vs 지원/임시 인력)?
- (Q3) 날짜 추가 후보 산출 규칙 = "학회 start 전후 ±N 평일"인지, 자유 date picker인지. (시안은 후보 메뉴로 단순화)
- (Q4) 출장 20,000 단가가 박수(泊) 기준이면, 셀 단일선택으로 "1박"만 표현됨 → 2박 이상 케이스 존재 여부 확인 필요(연 8일 규모상 드묾 가정).
