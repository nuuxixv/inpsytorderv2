# 07. 로드맵 — 학회 D-4개월

> 오늘은 2026-05-11이다. 하반기 학회 시즌(통상 9월 중순부터)을 가정하면 D-4개월. 이 4개월을 다섯 마일스톤으로 자른다. 일정이 길어 보이지만, 실제로 손에 쥐는 작업은 매 마일스톤 2~3주짜리다.

## 어떻게 나누는가

원칙 → 토큰 → 시안 → 코드 → 검수. 순서는 거꾸로 가지 않는다. 토큰 없이 시안 만들면 시안마다 새 색이 끼고, 시안 없이 코드 만들면 만든 다음 다시 깔아엎는다. 우리는 한 번 만든 걸 두 번 만들 시간이 없다.

각 마일스톤은 종료 시점에 건우님 승인이 있어야 다음으로 넘어간다(D2 결정).

## M1 — 원칙·토큰 확정 (5월 중순~5월 말, 2주)

**목표.** 01·02 문서를 시안 작업에 그대로 쓸 수 있는 수준까지 가져온다.

**할 일.**
- 01 원칙 문서를 건우님과 함께 한 번 더 읽고, 어색한 부분 다듬기
- 02 토큰의 정확한 헥사값·픽셀값을 정하고 Appendix_Spec.md 채우기
- 다크·고대비 세이프가드 토큰 보정 로직 초안(09 문서)

**종료 조건.** 토큰 카테고리 6종(brand·status·surface·text·shadow·primitives)의 기본값이 모두 정해진다. 시안 작업자가 의문 없이 쓸 수 있는 상태.

## M2 — 어드민 주문 관리 시안 (6월 초~6월 중순, 2주)

**목표.** A1(리스트), A2(상세 드로어) 시안 확정.

**할 일.**
- 04 프롬프트의 A1·A2를 시안 도구에 넣어 후보 3개씩 생성
- 게이트 1·2 통과
- 토큰 첫 적용 시안이라 토큰 자체에 반영해야 할 게 나오면 02로 피드백
- 학회 운영자 1인에게 시안 보여주고 의견 수집

**종료 조건.** A1·A2 시안 1종씩 확정. 코드 작업 들어갈 수 있는 상태.

## M3 — 어드민 코드 적용 + 나머지 시안 (6월 말~7월 중순, 3주)

**목표.** A1·A2 코드 적용, A3~A7 + S1 시안 동시 작업.

**할 일.**
- A1·A2 코드 적용 (기존 OrderManagementPage 교체 또는 점진 마이그레이션)
- A3·A4·A5·A6·A7·S1 시안 후보 생성 및 게이트 1·2 통과
- 게이트 3 실기기 QA 1차 (A1·A2 한정)

**종료 조건.** 어드민 주문 관리가 새 시스템으로 작동. 나머지 어드민 시안 확정.

## M4 — 어드민 전면 적용 + 고객 시안 (7월 말~8월 중순, 3주)

**목표.** 어드민 7화면 + 셸 코드 적용, 고객 화면 C1~C4 시안 확정.

**할 일.**
- A3~A7, S1 코드 적용
- C1·C2·C3·C4 시안 후보 생성 및 게이트 1·2 통과
- 게이트 3 실기기 QA 어드민 전 화면 PASS

**종료 조건.** 어드민이 새 디자인 시스템으로 완전 전환. 고객 화면 시안 확정.

## M5 — 고객 코드 적용 + 학회 freeze (8월 말~9월 초, 2주)

**목표.** 고객 화면 코드 적용 후 학회 시작 D-3까지 freeze.

**할 일.**
- C1~C4 코드 적용 (OrderPage, OrderStatusPage)
- 게이트 3 실기기 QA 고객 화면 PASS
- 학회 D-3부터 freeze, 회고 준비

**종료 조건.** 하반기 학회를 새 디자인 시스템으로 맞이한다.

## 변수 두 개

- **체리픽 작업.** 별도 트랙으로 진행 중(CTO에 위임). 이쪽은 디자인 시스템과 무관하게 main에 비-디자인 개선만 흡수. 디자인 작업과 충돌하면 디자인 우선.
- **알림톡 모바일 이슈.** PC Chrome 외에서 result_code 101. 디자인 작업과 별개로 진행. 학회 전 해결 필수지만 디자인 마일스톤을 지연시키지 않는다.
