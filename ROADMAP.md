## 🔴 우선순위 높음

### [연계 주문] parent_order_id — 추가 구매 플로우
**배경:**
학회 현장에서 고객이 1차 결제 후 추가 구매를 원하는 경우가 빈번하게 발생함.
현재는 각 주문이 독립적으로 생성되어 출고 시 수동으로 묶어야 하는 번거로움 존재.

**목표:**
- `orders` 테이블에 `parent_order_id uuid REFERENCES orders(id)` 컬럼 추가
- 어드민 주문관리에서 "연계 주문으로 묶기" 기능 제공
- 배송비 자동 재계산 로직:
  - 1차 + 2차 주문의 상품금액 합산이 3만원 이상이면 → 2차 주문 delivery_fee = 0
  - 1차에서 이미 배송비를 납부한 경우 2차 결제금액에서 배송비 차감
- 출고현황에서 연계된 주문을 묶음으로 표시 (같은 parent_order_id 그룹핑)

**DB 변경:**
```sql
ALTER TABLE orders ADD COLUMN parent_order_id uuid REFERENCES orders(id) ON DELETE SET NULL;
CREATE INDEX idx_orders_parent_order_id ON orders(parent_order_id);
```

**예상 공수:** 3~5일
**담당:** 마스터 개발자

## 🟡 중간 우선순위

### [이메일 알림] 주문 접수 확인 이메일
**배경:**
고객이 주문 완료 후 페이지를 닫아버리면 주문 내역 URL을 다시 찾기 어려움.
주문 접수 즉시 이메일로 토큰 URL을 발송하는 것이 가장 유지관리 부담이 적은 무료 솔루션.

**목표:**
- `create-order` Edge Function에서 Resend API 호출로 접수 확인 이메일 발송
- 이메일 내용: 고객명, 최종 결제금액, "주문 내역 확인하기" 버튼 (토큰 URL)
- Supabase 환경변수 `FRONTEND_URL` 설정 필요

**현황:** 코드 작성 완료 (`create-order/index.ts`), Resend 도메인 인증 후 활성화 예정

---

### [주문 조회] 고객용 주문 상태 페이지
**배경:**
이메일의 "주문 내역 확인하기" 버튼이 연결될 페이지.
로그인 없이 토큰(order UUID)으로 접근 가능한 읽기 전용 페이지.

**목표:**
- 라우트: `/order/status/:orderId`
- 표시 정보: 주문 상품 목록, 주문 상태, 최종 결제금액
- 상태가 변경되면 동일 URL에서 실시간 반영

## 🟢 낮은 우선순위

### [주문 조회 페이지] 이름+연락처 조회 (Fallback)
이메일도 없고 URL도 잃어버린 경우의 최후 수단.
- OTP 인증 (이름+연락처 입력 → 해당 번호로 SMS OTP → 주문 목록 노출) 검토
