# 인싸이트 현장주문 — 개발 로드맵

> 마지막 업데이트: 2026-04-09

---

## 1. 서비스 개요

**인싸이트 현장주문**은 의료 학회 현장 도서·검사 판매 전용 주문 관리 시스템이다.

- 운영 규모: 연 8일(학회 기간 한정), 하루 최대 100건, 연 ~800건
- 고객 플로우: 모바일 주문서 → 부스 카드단말기 결제 → 학회 종료 후 일괄 배송
- 학회 종료 후 알림: 카카오 알림톡(결제완료 시 자동 발송)만. 배송 추적 없음
- 어드민 운영자: 갤럭시 탭/아이패드 사용, 학회 현장에서 실시간으로 주문 관리

---

## 2. 현재 시스템 구성

### 2-1. 고객 사이드 (프론트엔드)

| 라우트 | 설명 |
|--------|------|
| `/order?events=<slug>` | 학회 전용 주문서. 슬러그 없거나 기간 외 접속 시 차단. |
| `/order/status/:token` | 주문 조회 페이지. UUID access_token 기반, 로그인 불필요. |

**주문서 흐름 (3단계):**
1. 상품 선택 — 검색창 + 인기/신규/카테고리 필터 + 그리드 카드 + 장바구니
2. 배송 정보 입력 — 이름/연락처/주소(다음 우편번호)/인싸이트ID/요청사항
3. 최종 확인 + 제출 — 할인율, 배송비(정가 기준 3만원 이상 무료), 배송 예정일 표시

**주문 제출:** Supabase Edge Function `create-order` 호출. 완료 시 `access_token`으로 주문 조회 페이지로 이동.

**알림톡:** 결제완료(`paid`) 상태 전환 시 브라우저에서 원샷 msgagent API 직접 호출. PC Chrome에서 정상 동작. iOS Safari에서 result_code 101 발생(미해결, msgagent 측 문의 필요).

---

### 2-2. 어드민 사이드

**접근:** `/admin/*` — Supabase Auth 기반 로그인. RBAC 권한 체계.

**권한 레벨:**

| 레벨 | 설명 |
|------|------|
| `master` | 모든 기능 접근. 설정 페이지 포함. |
| `orders:view` / `orders:edit` | 주문 조회/편집 |
| `events:view` / `events:edit` | 학회 관리 |
| `products:view` / `products:edit` | 상품 관리 |
| `dashboard:view` | 대시보드 |
| `users:manage` | 사용자 계정 관리 |

---

#### 페이지별 현황

**대시보드 (`/admin/dashboard`)**
- 연도/학회/상세 행사 3단계 계층 필터 (각각 드롭다운, 학회·행사는 체크박스 멀티 선택)
- 학회 필터: `societies` 테이블과 동기화
- KPI 4종: 검사 판매액, 도서 판매액, 배송비, 총 매출액 (취소/환불 제외, 배송비 분리)
- YoY 전년 대비 증감률 (연도 단일 선택 + 전체 행사 합산 시 노출)
- 검사/도서 판매 순위 Top5
- 오늘 주문 수 + 주문 상태 분포 바
- 최근 주문 5건 (클릭 시 상세 모달)
- Realtime: 신규 주문 INSERT 시 상단 알림 토스트 발생 (Supabase Realtime)

**주문 관리 (`/admin/orders`)**
- 필터: 학회 멀티 선택, 주문 상태 멀티 선택, 고객명 검색, 날짜 범위 (기본: 최근 30일), 초기화 버튼
- URL 파라미터로 상태 필터 프리셋 가능 (`?status=paid` 등)
- 페이지네이션 (기본 20건/페이지)
- 체크박스 다중 선택 + 일괄 상태 변경
- 주문 상세 모달: 정보 편집, 상품 항목 편집, 상태 변경, 알림톡 재발송, 연계 주문 연결/해제
- 연계 주문 (parent_order_id): 합배송 처리용. 2차 주문의 배송비 자동 조정.
- 신규 주문 추가 (NewOrderModal): 행사 선택 → 상품 검색/장바구니 → 고객 정보 → 저장
- 엑셀 다운로드: 도서 전용 / 검사 전용 / 전체 통합 3종

**학회 관리 (`/admin/events`)**
- 학회(societies) 관리: societies 테이블 CRUD (별도 다이얼로그)
- 행사 CRUD: 이름, 슬러그, 기간, 할인율, 배송 예정일, 주최 학회, 연도, 시즌
- 행사 상태 자동 분류: 예정/진행중/종료
- 주문 URL 복사 버튼 (`/order?events=<slug>`)

**상품 관리 (`/admin/products`)**
- 상품 CRUD: 이름, 가격(정가), 카테고리(도서/검사/온라인코드 등), 태그(학회 연동)
- 플래그: is_popular(인기), is_new(신규), is_discountable(할인 적용 여부)
- 상품별 태그로 학회 연동 (해당 학회 주문서에서만 노출)
- 엑셀 업로드/다운로드 (product_code 기준 upsert)
- 체크박스 다중 선택 + 선택 삭제 + 전체삭제(받아쓰기 확인)
- 선택 상품 일괄편집 모달 (is_popular / is_new / tags)

**출고 현황 (`/admin/fulfillment`)**
- 결제완료 주문 전용 뷰 (paid, preparing, completed 상태)
- 행사별 필터, 도서/검사/전체 토글
- 연계 주문 병합 표시 (합배송 그룹)
- 개별 주문 상태 변경
- 배송지 클릭 복사: 도로명주소 / 상세주소 분리 복사 (우편번호 제외)

**사용자 관리 (`/admin/users`)**
- 계정 생성/삭제, 권한 부여/해제
- 비밀번호 초기화

**설정 (`/admin/settings`)**
- 배송비 기준금액 (무료배송 임계값)
- 배송비 단가
- master 권한만 접근

---

### 2-3. DB / 인프라

- **Supabase**: PostgreSQL + Auth + Realtime + Edge Functions + Storage
- **Vercel**: 프론트엔드 배포 (SPA rewrite)
- **RLS**: anon SELECT 완전 차단, get_order_by_token RPC로 토큰 기반 단건 조회만 허용
- **Edge Functions**: `create-order` (주문 생성 + access_token 발급)

## 4. 완료된 주요 작업 목록

- 주문 상태 6단계 (pending/paid/preparing/shipped/completed/cancelled/refunded)
- RBAC 권한 체계 + RLS 보안 강화 (anon 차단, get_order_by_token RPC)
- UUID access_token 기반 주문 조회 (순번 ID 열거 공격 차단)
- 카카오 알림톡 연동 (결제완료 자동 발송, 어드민 재발송 버튼)
- 이메일 수집 제거 (알림톡으로 대체)
- 주문 수정 RPC 레거시 email 참조 제거 (`update_order_details`, `update_order_with_items`)
- 연계 주문 (parent_order_id 합배송, 배송비 자동 조정)
- 배송 예정일 (events.estimated_delivery_date)
- 출고현황 결제완료 전용 뷰
- 출고현황 배송지 복사 UX 개선 (도로명/상세주소 분리 복사)
- 어드민 대시보드 매출 계산 로직 (배송비 분리, 취소/환불 제외, YoY)
- 대시보드 학회 필터 → societies 테이블 동기화
- 대시보드·주문관리 행사/상태 필터 → 체크박스 멀티 선택
- 신규 주문 모달 → 상품 검색/장바구니 방식으로 재설계
- 상품 엑셀 업로드 boolean 파싱 보강 (TRUE 문자열 + boolean true 대응)
- 상품 관리 다중 선택/선택 삭제/전체삭제(받아쓰기) 구현
- 상품 관리 일괄편집 모달 구현 (인기/신상품/태그)
- 사이드바/페이지 타이틀 아이콘 일관성 수정
- 어드민 타이틀 "인싸이트 현장주문"으로 통일
- 주문 검색 연락처 하이픈 무관 검색
