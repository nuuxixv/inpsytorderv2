## 9. ⚙️ 데이터베이스 내부 톱니바퀴 (SQL Functions & Triggers)

프론트엔드나 엣지 함수(Edge Function)가 데이터를 조작하기도 전에, DB 창고 그 자체에 내장시켜버린 '자동화 로직'들입니다. 외부 인터넷이 끊기든 서버가 죽든, DB 안에서 안전하게 돌아가는 가장 견고한 처리 방식입니다.

### 📌 9-1. update_order_with_transaction.sql (트랜잭션: 올 오어 나띵)

어드민 화면에서 사장님이 '주문 내역'이나 '품목'을 수정하고 저장할 때 작동하는 전용 함수(`update_order_with_items`)입니다.
**[왜 이렇게 짰을까?]** 
주문 정보를 고치고 나서, 바뀐 품목 정보를 고치려는 찰나에 서버가 뻗으면 어떻게 될까요? 결제 금액은 5만 원으로 줄었는데, 품목은 옛날 10만 원어치가 그대로 남아 구멍이 나는 대참사가 일어납니다.
이 함수는 **"고칠 거면 한 폭의 그림처럼 동시에 다 고치든지, 중간에 삑사리 나면 아예 안 고친 걸로 되돌려 놔!" (Transaction 패턴)** 전략을 씁니다. 기존 장바구니(`order_items`) 내용을 싹 다 지운 뒤 새 내용으로 한꺼번에 밀어 넣습니다. 

### 📌 9-2. create_paid_order_trigger.sql (자동 알림 방아쇠)

말 그대로 사격장에 있는 '방아쇠(Trigger)'입니다. 프론트엔드나 엣지 함수가 "알림 보내!"라고 명령하지 않아도, DB 창고가 실시간으로 수문장처럼 지켜보다가 자기 스스로 총을 쏩니다.
- **감시 조건:** 특정 주문서(`orders` 테이블)의 상태(`status`)가 방금 전에는 '결제 전'이었는데, 업데이트를 통해 지금 막 **'결제 완료(paid)'**로 확실히 바뀌었는가?
- **격발 행동:** 조건이 맞으면 `notify_order_paid()` 함수가 실행되어 매장 전역에 `order_paid`라는 사이렌(pg_notify)을 울립니다.
- *(이 사이렌 소리를 듣고 다른 자동화 봇(Edge Function)이 깨어나 "결제완료 이메일을 발송해라!" 하고 움직이게 만드는 고도의 백엔드 아키텍처 패턴입니다.)*

3. 모니터링을 방해하는 기만적인 **가짜 성공(200 OK) 방어코드 교체**

이 네 가지는 **<Inpsyt Order V2 엔터프라이즈 전환 리팩토링>**이라는 다음 여정의 완벽한 등대(기획서)가 되었습니다. 
이 4가지 수술 중, 가장 먼저 칼을 대고 싶은(우선순위 1번) 파트부터 지시해 주시면 당장 코딩에 착수하겠습니다!

🛤️ 2. 스터디 (각주 달기) 추천 순서
코드는 쉬운 것(단순 조회)부터 시작해서 어려운 것(복잡한 화면 ➡️ 백엔드)으로 나아가야 흥미를 잃지 않습니다.

1단계: 워밍업 (API 통신 원리)
src/api/products.js
학습 포인트: async / await (비동기 통신), try-catch (에러 잡기), Supabase에서 데이터를 어떻게 가져오는지 등 가장 기초적인 자바스크립트 통신 문법을 배웁니다.
2단계: 화면과 상태의 기초 (React State)

CustomerInfoStep.jsx
학습 포인트: 사용자가 글씨를 치면 화면이 어떻게 그걸 기억하는지(useState), 버튼을 누르면 어떻게 함수가 실행되는지 배웁니다.
3단계: 화면의 흐름과 생명주기 (React Hooks)

ProductSelectionStep.jsx
학습 포인트: 화면이 켜지자마자 무언가를 실행하는 법(useEffect), 0.3초를 기다려주는 최적화(Debounce) 등 프론트엔드의 꽃을 배웁니다.
4단계: 중간보스 (God Component)

OrderPage.jsx
학습 포인트: 여러 개의 자식 화면(Step)들을 부모가 어떻게 통제하고 데이터를 끌어모으는지(Props) 전체적인 그림을 스터디합니다.
5단계: 진최종보스 (Backend Edge Function)
create-order/index.ts
학습 포인트: 서버 컴퓨터(Deno/Node.js) 안에서 코드가 어떻게 돌아가는지, 보안과 트랜잭션을 코드로 어떻게 짜는지 분석합니다.
📝 3. 스터디를 위한 '마법의 프롬프트' (복붙용)
앞으로 특정 파일을 여시고 코딩 AI(Cursor, Cline, 혹은 저)에게 스터디를 요청하실 때 이 프롬프트를 복사해서 붙여넣으세요.

[코드 라인바이라인 각주 스터디 프롬프트]

"나는 비개발자 기획자이고, 지금부터 내 시스템의 코드를 라인 단위로 통제하기 위해 스터디를 할 거야. 현재 열려있는 이 코드를 수정하지 말고, 코드가 어떻게 굴러가는지 핵심 로직마다 한글로 주석(//)을 상세하게 달아줘.

[주석 작성 규칙]

단순 번역이 아니라, **'왜 이렇게 짰는지(기획적 의도)'**와 **'어떤 순서로 실행되는지'**를 비유를 들어 설명해 줄 것.
useState, useEffect, async/await, map, filter 같은 React나 JS의 뼈대가 되는 예약어/문법이 나오면, 그게 무슨 역할을 하는지 선생님처럼 각주로 달아줄 것.
주석 처리된 완성본 코드를 출력해주면, 역으로 내가 질문하면서 스터디를 진행할게."
